## 分層架構

```
層級          模組                    職責
------------------------------------------------
API 層       api/                    處理請求/回應
服務層       services/               業務邏輯
資料層       models/, repositories/  資料存取
工具層       utils/                  共用工具
```

### 呼叫規則

- 上層可呼叫下層
- 下層不可呼叫上層
- 同層盡量不互相呼叫

## 新增模組規則

1. 確認沒有現有模組可擴充
2. 放在對應的分層目錄
3. 更新本文件
4. 不建立重複功能
