# 技術選型說明

版本：1.0.0
最後更新：2025-12-31

---

## 概述

本專案為瀏覽器分頁與書籤管理插件，採用 Manifest V3 架構，結合現代前端技術棧打造流暢的桌面應用級使用體驗。

---

## 前端技術

### 選用方案

| 項目 | 選擇 | 說明 |
|-----|------|------|
| 框架 | React 19 | 生態系對 Extension 支援最佳，Hooks 開發效率高 |
| 樣式 | Tailwind CSS | 深色模式處理高效，原子化 CSS 開發迅速 |
| 建置工具 | Vite | 極速 HMR，搭配 CRXJS 簡化 Extension 開發 |
| 圖示 | Lucide React | 輕量、風格一致、支援 Tree-shaking |

### 選擇理由

- React 生態對 Chrome Extension 開發有成熟支援（CRXJS、Plasmo 等）
- Tailwind CSS 的 Dark Mode 切換僅需 `dark:` prefix，符合截圖中深色 UI 需求
- Vite 搭配 CRXJS Vite Plugin 可實現 Extension 即時熱更新
- Lucide React 圖示簡潔、一致性高，與參考設計風格契合

### 風險評估

| 風險 | 可能性 | 影響 | 因應方式 |
|-----|:-----:|:----:|---------|
| Manifest V3 政策變動 | 低 | 中 | 追蹤 Chrome 官方更新 |
| CRXJS 維護停滯 | 低 | 中 | 備選 Plasmo 或 vite-plugin-web-extension |

---

## 後端技術

### 選用方案

| 項目 | 選擇 | 說明 |
|-----|------|------|
| 語言 | 無（純前端） | MVP 階段資料存於本地 |
| 框架 | 無 | 使用 Chrome Storage API |
| 資料庫 | Supabase（未來） | 雲端同步階段啟用 |

### 選擇理由

- MVP 階段無需後端，使用瀏覽器原生 Storage API 儲存書籤與設定
- 未來需要雲端同步時，Supabase 提供即時訂閱與 Auth 功能

### 風險評估

| 風險 | 可能性 | 影響 | 因應方式 |
|-----|:-----:|:----:|---------|
| 本地儲存空間限制 | 低 | 低 | chrome.storage.sync 配額 100KB 足夠 |
| 資料遺失（未同步） | 中 | 中 | 提供手動匯出/匯入功能 |
